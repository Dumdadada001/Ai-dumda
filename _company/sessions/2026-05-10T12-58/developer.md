# 🛠️ Python 오류 진단 및 문제 해결 가이드라인 (체크리스트)

## 🎯 Step 0: 필수 정보 제공 요청 (가장 중요)
어떤 경우에도 이 세 가지 핵심 정보를 먼저 복사하여 붙여넣는 것이 최우선입니다. 이 데이터 없이는 추측에 그칩니다.

1.  **❌ 정확한 에러 메시지 (Traceback):** 코드를 실행했을 때 터미널에 출력되는 오류 전체 텍스트(빨간색 부분)를 통째로 복사하여 붙여넣어 주세요.
    *(예: `NameError: name 'variable_name' is not defined`)*
2.  **💻 발생 환경 정보:** 코드를 실행하는 환경의 사양을 알려주세요.
    *   운영체제(OS): (예: Windows 10, macOS Sonoma, Ubuntu 22.04 등)
    *   파이썬 버전: `python --version` 명령어를 실행한 결과값. (예: Python 3.11.5)
3.  **📄 문제 코드 일부:** 오류가 발생했다고 의심되는 **최소 단위의 실행 코드 블록(Snippet)**을 복사하여 붙여넣어 주세요.

## 🔬 Step 1: 흔하게 발생하는 오류 유형별 체크포인트

만약 에러 메시지 자체가 없다면, 아래 세 가지 주요 오류 유형 중 어느 것과 유사한 증상인지 확인해 보세요.

### A. 환경 설정 및 라이브러리 누락 관련 (Dependency Issues)
*   **🚨 예상되는 에러:** `ImportError: No module named 'xxx'` 또는 `ModuleNotFoundError: No module named 'xxx'`
*   **✅ 체크포인트:** 이 코드가 외부 기능을 사용하려면 해당 기능이 파이썬 환경에 설치되어 있어야 합니다.
    1.  **설치 여부 확인:** 필요한 라이브러리(예: pandas, requests 등)가 모두 설치되었는지 확인합니다.
    2.  **재설치 시도:** `pip install [라이브러리명]` 명령어를 사용하여 재설치를 시도해봅니다. (가상 환경 사용을 강력히 권장합니다.)

### B. 문법적 실수 관련 (Syntax Errors)
*   **🚨 예상되는 에러:** `SyntaxError: invalid syntax` 또는 `NameError`
*   **✅ 체크포인트:** 파이썬 언어의 규칙(문법)을 따르지 않았거나, 변수/함수를 잘못 정의했을 가능성이 높습니다.
    1.  **콜론(:) 확인:** `if`, `for`, `def`, `class` 등의 키워드 뒤에 콜론(`:`)이 빠지지 않았는지 확인합니다.
    2.  **들여쓰기(Indentation):** 파이썬은 들여쓰기로 코드 블록을 구분합니다. 탭(Tab)과 스페이스바(Space)가 혼용되지 않도록 **일관성 있게 공백 4칸**으로 통일되었는지 확인하세요. (가장 흔한 실수 중 하나입니다.)
    3.  **변수명 오타:** 변수 이름이나 함수 이름을 정의할 때 사용했던 철자 그대로 호출하고 있는지 다시 한번 확인합니다.

### C. 실행 흐름 및 논리적 오류 관련 (Runtime/Logic Errors)
*   **🚨 예상되는 에러:** `TypeError`, `KeyError`, `IndexError` 등
*   **✅ 체크포인트:** 코드는 문법적으로 완벽하지만, 프로그램이 기대한 방식대로 데이터를 처리하지 못할 때 발생합니다.
    1.  **데이터 타입 확인:** 변수에 담긴 데이터가 예상했던 자료형(String, Integer, List 등)인지 확인하세요. (예: 숫자로 계산해야 하는데 문자열로 인식된 경우 `TypeError` 발생).
    2.  **인덱스 범위를 벗어났는지 확인:** 리스트나 배열의 크기보다 큰 인덱스를 참조하려고 했는지 (`IndexError`) 확인합니다.
    3.  **값 Null 체크:** 함수가 값을 반환하는 지점에서 값이 `None`이거나 비어있을 수 있습니다. 이 경우에 대한 예외 처리(try-except)를 추가해야 합니다.

## 🚀 Step 2: 종합 진단 (궁극의 해결책)
위 모든 단계를 거쳐도 원인을 찾기 어렵다면, 코드를 실행하는 **가상 환경(Virtual Environment)**을 새로 만들고 필요한 라이브러리만 설치한 뒤 다시 시도해 보세요. 이는 환경적 충돌로 인한 오류를 90% 이상 제거할 수 있는 가장 강력한 방법입니다.
